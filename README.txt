情報科学科教育プログラム希望調査システムの実用開発

1.目的
Webプログラミング演習で開発した希望登録システムを実用化にむけてさらに開発する。
  
2. 機能要件
2.1. 調査期間の設定機能
システム設定機能として、調査期間を設定し、期間前や期限が過ぎた後は回答できないようにする。

2.2.配属結果確認機能
ログイン中の学生の配属結果を確認する機能。本人以外の配属結果を閲覧できないものとする。

2.3.希望状況一覧機能（未提出者も含む）
希望を提出した学生全員の学籍番号、氏名、取得単位数、GPA、希望教育プログラム、配属結果を含む希望状況を学籍番号順で並べ替えて出力する機能。
未提出者も含まれる。

2.4.配属決定機能
員が学生の配属先の教育プログラムを決定する機能。希望状況一覧画面の各行に「配属決定」ボタンを設け、クリックすると、当該学生の配属決定画面に遷移し、そこで配属決定を行う。希望者の学籍番号、氏名、取得単位数、GPA,希望教育プログラム、登録要件を表示しながらその学生の配属先の教育プログラムを決定する。なお、配属は自動的に条件を満たす教育プログラムに決定するのではなく、教員の判断でどの教育プログラムにも決定できるものとする。

2.5.未提出者一覧機能
教育プログラム希望を提出していない学生の学籍番号、氏名、取得単位数、GPAを一覧する機能。

2.6.希望状況集計機能
教育プログラム別の希望者数を集計して、集計結果(プログラム名、希望者数、決定者数)を表示する機能。０名の場合も含む。
